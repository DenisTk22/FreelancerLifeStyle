# Marging и Padding. 
Ссылка на урок https://www.youtube.com/watch?v=uSGKPYEPY9M&list=PLM6XATa8CAG4F9nAIYNS5oAiPotxwLFIr&index=30
# Применяются к блочным и блочно-строчным объектам
- При **отсутствии** свойства box-sizing: border-box; padding будет увеличивать размер облока, marging также увеличит сам блок.
- При **наличии** свойства box-sizing: border-box; padding будет входить в размер облока, marging увеличит сам блок.
- padding не разрывает заливки блоков на странице, т.к. расширяет сами блоки.
- marging разрывает заливки блоков, т.к. раздвигает сами блоки.
- Эффект галочки у текста, можно добиться, задав галочку фоновым изображением и сделать Отступ padding, отодвинув текст от галочки.
- Отступы и позиционирование.
    - Если относительно объекта применяется абсолютное позиционирование, то можно разделить этот объект и, например, текст, применив к родителю padding;
- Отступы и overflow: hidden;
    - С помощью внутреннего отступа padding можно отобразить скрытую часть объекта.
- Центровка блока.
    - С помощью внешнего отступа margin можно отцентрировать блочный объект с заданным ограничением ширины:
        - margin: 0px auto;
- Отрицательное значение margin.
    - При отрицательном margin можно надвинуть второй объект на первый, а также скрыть содержимое первого объекта под вторым, и также третий объект подтянится за вторым:
        - .class {
        position: relative;
        z-index: 2;
        marging: -50px 0 0 0;}
    - Можно использовать для устранения padding для крайних блоков.
        - При задании padding для блоков, крайние блоки тоже будут отдалены от края, это можно устранить задав общему блоку отрицательный margin, например, слева и справа на ту же величину. Будет эффект gap.
    - Комбинация с padding.
        - При исползовании padding для отображения скрытой части объекта, увеличится размеры отступов. Для устранения данного эффекта можно использовать отрицателный margin на то же значение.
        - .slider {
        display: flex;
        overflow: hidden;
        margin: -25px 0;
        padding: 25px 0;}
- Слопывание margin.
    - когда оба значения margin положительные, итоговый размер поля будет равен **большему** значению.
    - если одно из значений отрицательное, то для вычисления размера поля нужно получить сумму размеров.
    - если оба значения отрицательные, сравниваются модули этих чисел и выбирается **большее** по модулю.
    - если значения указаны в разных единицах CSS, они приводятся к одному и выбирается **большее** значение.
    - если у потомка поле margin больше чем у родителя, то приоритет отдается потомку.

- **Принципы применения отступов**
    - отступы следует задавать от предыдущего элемента к следующему: сверху вниз / слева направо.
    - у последнего элемента группы отступ следует обнулить, с помощью псевдокласса last-child, чтобы он не складывался с отступом родителя.
    - отступы задаются самому верхнему родителю в конструкции. 
        - Например, есть список, список вложен в отдельный блок. Отстун, например, margin следует задать отдельному блоку.
    - внешние отступы не следует задавать для независимых объектов (БЭМ блок), которые могут быть использованы в других блоках, т.к. они принесут отступы с собой, где это не нужно. В таких случаях отступ следует задавать дополнительном индивидуальном классе.
    - отступы должны улучшать удобство пользования и восприятия.
        - не нужно делать слишком большие и лишние отступы.
    - увеличивать площать нажатия на элемент-ссылку при помощи padding, для удобства.
